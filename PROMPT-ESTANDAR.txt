Estoy trabajando en un proyecto Angular standalone con los siguientes
estándares obligatorios que SIEMPRE debes respetar:

## Change Detection — REGLAS ABSOLUTAS
- Todos los componentes (páginas Y shared) deben usar:
  changeDetection: ChangeDetectionStrategy.OnPush
- Después de cualquier cambio async (Observable, Promise) usar SOLO:
  this.cdr.markForCheck()
- NUNCA usar: this.cdr.detectChanges()
- NUNCA usar: NgZone / zone.run()
- NUNCA usar: setTimeout() para forzar renders

## Estructura de un componente de página
- Siempre inyectar ChangeDetectorRef en el constructor
- NUNCA inyectar NgZone
- Patrón obligatorio en todo subscribe():
  next: (data) => { this.datos = data; this.cdr.markForCheck(); }
  error: ()     => { this.loading = false; this.cdr.markForCheck(); }

## Razón
Los componentes shared del proyecto (app-perfil-select, app-cuenta-search,
app-empleado-search, app-select, app-paginator) usan OnPush. Si un componente
padre usa Default, Angular no propaga los cambios correctamente y la vista
no refresca hasta el próximo evento del usuario.

SEGUNDO FIX, cuando no cargan los registros

El fix específico fue ApplicationRef.tick().
La diferencia con los casos anteriores:

Casos anteriores (rend-m, rend-d, perfiles) — el problema era que el propio componente con OnPush no se actualizaba. El fix era form.statusChanges.subscribe(() => cdr.markForCheck()) o ChangeDetectionStrategy.Default en el componente mismo.
Este caso — el problema era diferente: el usuario-search (hijo) cargaba datos HTTP y llamaba cdr.markForCheck() sobre sí mismo, pero eso solo marca el hijo para revisión. El padre users.component tiene OnPush y no sabe que su hijo cambió, entonces no re-renderiza el árbol. markForCheck() sube por el árbol marcando ancestros, pero en algunos casos con componentes standalone y lazy el árbol no está completamente conectado.
appRef.tick() — dispara un ciclo de detección completo desde la raíz, igual que cuando Angular detecta un evento del navegador. No importa si el padre es OnPush o no, todos se revisan.
